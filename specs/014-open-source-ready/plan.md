## Key Design Decisions

1. **Version from git tag**: The publish workflow strips the `v` prefix from the tag and passes it to `dotnet pack` via `/p:Version=`. No changes to Directory.Build.props needed — the workflow overrides at pack time.

2. **Single CI runner (ubuntu-latest)**: Cross-platform CI is deferred. The existing test suite targets .NET 8 which is cross-platform, and ubuntu-latest is the fastest/cheapest runner.

3. **No license headers in source files**: MIT license at repo root is standard practice. Adding headers to 400+ files would be noisy and adds maintenance burden.

4. **README structure follows Testimize pattern**: Progressive disclosure — value prop first, installation second, deep features third, references last.

5. **Docs already complete**: All 16 docs files are substantive (58-430 lines each). No new docs need to be written — just ensure README links point to them correctly.
