# Document Management

## Overview

The system manages documents attached to citizen applications and internal administrative processes. Documents flow through a lifecycle from upload to archival, with versioning, digital signatures, and retention policies.

## Document Upload

### Citizen Uploads

Citizens can upload documents as part of their applications. Supported formats and limits:

| Format | Max Size | Notes |
|---|---|---|
| PDF | 20 MB | Preferred for official documents |
| JPEG, PNG | 10 MB | Photos of documents, auto-converted to PDF after OCR |
| DOCX | 15 MB | Converted to PDF upon upload for consistent viewing |
| HEIC | 10 MB | Converted to JPEG then to PDF (iPhone photos) |

Upload constraints:
- Maximum 10 files per application
- Maximum total size per application: 50 MB
- File names are sanitized: special characters replaced with underscores, maximum 100 characters
- If a file name already exists in the application, a suffix is added: `document.pdf` → `document_2.pdf`
- Virus scan is performed on every upload. Infected files are rejected with: "This file could not be uploaded for security reasons."
- Password-protected PDFs are rejected: "Please upload an unprotected version of this document."

### Internal Document Generation

The system generates documents from templates:

- **Templates** are DOCX files with placeholder tokens: `{{citizen.firstName}}`, `{{application.referenceNumber}}`, `{{currentDate:DD.MM.YYYY}}`
- Templates are versioned. When a template is updated, existing generated documents retain their original template version
- Generated documents are converted to PDF/A-2b before delivery to citizens
- Each generated document receives a unique barcode (Code 128) and QR code containing the document verification URL

## Document Versioning

Every document has a version history:

- **Major version**: Created when the document content changes (e.g., new upload replaces old one)
- **Minor version**: Created when metadata changes (e.g., status change, comment added)
- Version format: v1.0, v1.1, v2.0, etc.
- Previous versions are retained and accessible to administrators but not to citizens (citizens always see the latest version)
- Deleting a document creates a "tombstone" record — the document is hidden but the version history and audit trail are preserved for 10 years

## Digital Signatures

### Qualified Electronic Signatures (QES)

Documents requiring legal validity must be signed with a qualified electronic signature:

- Supported providers: B-Trust, Evrotrust, InfoNotary, StampIT
- Signature validation checks: certificate chain, revocation status (OCSP/CRL), timestamp
- The system embeds a visible signature block in the PDF: signer name, timestamp, certificate issuer, and a "SIGNED" stamp
- If the certificate is expired at the time of signing, the signature is rejected
- If the certificate is revoked after signing but the signature included a qualified timestamp, the signature remains valid

### System Electronic Seal

The system applies its own qualified electronic seal to all generated documents. This seal:
- Confirms the document was generated by the system
- Includes a qualified timestamp from a trusted Time Stamping Authority (TSA)
- Is verified automatically when a citizen downloads the document — a green banner shows "Document authenticity verified"

### Signature Verification Service

A public endpoint allows anyone to verify a document's authenticity:
1. Upload the document or enter the document's unique code (from barcode/QR)
2. The system checks: electronic signatures/seals, document integrity (hash comparison), and whether the document exists in the system
3. Results: "Authentic" / "Modified after signing" / "Document not found in system"

## Document Status Workflow

```
DRAFT → PENDING_REVIEW → APPROVED → SIGNED → DELIVERED → ARCHIVED
                       → REJECTED → (citizen can resubmit → DRAFT)
```

- Only documents in DRAFT or REJECTED status can be edited
- APPROVED documents cannot be modified — a new version must be created
- SIGNED documents are immutable
- DELIVERED means the citizen has been notified and can download
- ARCHIVED is automatic after the retention period (based on document type: 1 year, 5 years, 10 years, or permanent)

## Search

Documents are searchable by:
- Application reference number
- Citizen name or EGN
- Document type
- Date range (uploaded/created/signed)
- Status
- Full-text search across OCR-extracted content (for scanned documents)

Search results are paginated (20 per page) and sortable by date, name, or relevance. Full-text search results highlight matching terms in the preview.

## Retention and Purge

| Document Type | Retention Period | After Expiry |
|---|---|---|
| Identity documents (copies) | 1 year after application closure | Permanently deleted (all versions, all storage) |
| Application attachments | 5 years after application closure | Moved to cold storage, accessible on request within 30 days |
| Generated certificates | 10 years | Moved to cold storage |
| Audit-related documents | 10 years | Moved to cold storage |
| Administrative decisions | Permanent | Never deleted |

Purge runs nightly at 02:00 UTC. A purge report is generated listing all deleted documents. Documents scheduled for purge are flagged 30 days in advance, allowing administrators to extend retention if needed.
