# Payment timeout handling and retry mechanism

## Preconditions

Citizen has submitted a service application and selected card payment. ePay.bg gateway is configured to simulate timeout (no response within 30 seconds).

## Steps

1. Submit a service application with a fee
2. Select 'Bank Card (Visa/Mastercard)' as payment method
3. Click 'Proceed to Payment'
4. Simulate ePay.bg gateway timeout (no response within 30 seconds)
5. Verify transaction is marked as PENDING
6. Verify citizen sees message: 'Payment is being processed. You will receive a confirmation email within 15 minutes.'
7. Wait 15 minutes (or simulate time passage)
8. Verify system queries ePay.bg for transaction status
9. Simulate ePay.bg confirms failure
10. Verify application returns to 'Awaiting Payment' status
11. Verify citizen can retry payment
12. Attempt payment again and verify retry is allowed

## Expected Result

When ePay.bg does not respond within 30 seconds, transaction is marked PENDING and citizen sees processing message. After 15 minutes without callback, system queries ePay.bg. If failure is confirmed, application returns to 'Awaiting Payment' status. Citizen can retry payment.

## Test Data

Service: Certificate of Marital Status, Fee: 5.00 BGN
