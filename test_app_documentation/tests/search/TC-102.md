# Search with invalid field-specific filter syntax

## Preconditions

User is authenticated and has access to search functionality

## Steps

1. Navigate to the global search bar
2. Enter search query with invalid field name: 'invalidfield:pending'
3. Press Enter to execute search
4. Observe the search behavior and results
5. Note any error messages or how the system handles the invalid syntax

## Expected Result

System handles invalid field-specific filter gracefully. Either treats the entire query as plain text search (searching for literal 'invalidfield:pending') or displays a helpful error message indicating the field is not recognized. Valid field-specific filters are: status, service, region, clerk, date, amount, egn, ref.

## Test Data

Query: 'invalidfield:pending'
