# Search results filtered by user permissions for clerk role

## Preconditions

User is authenticated as a Clerk role. Test data includes applications assigned to the clerk, applications in their department queue, and applications outside their department.

## Steps

1. Log in as a user with Clerk role
2. Navigate to global search
3. Execute a broad search query that would match applications across multiple departments
4. Review the search results
5. Verify that only applications assigned to the clerk or in their department queue are visible
6. Verify that applications from other departments are not visible in results
7. Check that result count reflects only permitted applications

## Expected Result

Search results are automatically filtered by clerk permissions. Only applications assigned to the clerk or in their department queue are displayed. Applications from other departments are excluded. Result counts are accurate and reflect permission-filtered results. Filtering happens at query level, not post-query.

## Test Data

Clerk user: clerk@example.com, Department: Sofia Regional Office
