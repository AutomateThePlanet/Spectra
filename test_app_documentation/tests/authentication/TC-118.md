# SSO auto-provisions new user from SAML assertion

## Preconditions

SSO is enabled, user exists in Azure AD but not in local system

## Steps

1. Navigate to login page with user that exists in Azure AD but not locally
2. Complete SSO authentication via Azure AD
3. Observe system behavior after successful SSO
4. Verify new user account is created
5. Check user role and department assignment

## Expected Result

If SSO authentication succeeds but user does not exist locally, a new account is provisioned automatically with 'Employee' role and department from SAML assertion. SSO_PROVISIONED event is logged. User is authenticated and can access the system.

## Test Data

Azure AD user not in local system, SAML assertion with department attribute
