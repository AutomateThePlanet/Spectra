# Password rejected when reusing one of last 5 passwords

## Preconditions

User has password history with at least 1 previous password

## Steps

1. Navigate to password change form
2. Enter a password that was used within the last 5 password changes
3. Ensure password meets all other complexity requirements
4. Submit the password

## Expected Result

Password is rejected with error message indicating password reuse policy. Password is not saved.

## Test Data

Previously used password from last 5 changes
